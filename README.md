# Проект: Авторизация и формы (ДЗ4)

## Как запустить

1. Склонировать репозиторий.
2. Заменить `.env.example` на `.env` со своими данными, например:
DB_NAME=django_db
DB_USER=german
DB_PASSWORD=qwerty
DB_HOST=db
DB_PORT=5432
SECRET_KEY=django-insecure-fixme-12345
DEBUG=True
3. В терминале выполнить: `docker-compose up --build`
4. Дождаться запуска контейнеров, затем:  
`docker-compose exec web python manage.py migrate`
5. Для заполнения тестовыми данными:  
`docker-compose exec web python manage.py fill_db 10000`
6. Открыть в браузере `http://localhost:8000`
7. Для остановки: `docker-compose down -v`

## Что реализовано

- **Авторизация и регистрация**  
Встроенная система Django. Регистрация с полями: логин, email, имя, пароль. Пароль проверяется стандартными валидаторами. При регистрации автоматически создаётся профиль. Вход поддерживает параметр `?next` (безопасный редирект).

- **Профиль**  
Доступен только авторизованным. Можно изменить email, никнейм и загрузить аватар.

- **Добавление вопроса** (`/ask/`)  
Авторизованный пользователь вводит заголовок, текст и теги через запятую. После сохранения перенаправляется на страницу вопроса. Для неавторизованных поля заблокированы с подсказкой.

- **Ответ на вопрос**  
На странице вопроса (`/question/{id}/`) авторизованный может написать ответ. После отправки остается та же страница. Гости видят неактивную форму.

- **Лайки вопросов**  
Кнопка лайка на списках и на странице вопроса. Один пользователь – один лайк. Пока что без JS и AJAX.

- **Формы**  
Валидация и сохранение вынесены в методы `clean()` / `save()` (разделение ответственности). Защита от CSRF. При ошибках введённые данные сохраняются.

- **Интерфейс для гостей**  
Элементы, недоступные неавторизованным (кнопки лайка, формы добавления вопроса/ответа), отключены (`disabled`) и имеют подсказки `title`.

- **Выход**  
Кнопка «Выйти» видна только авторизованным. После выхода пользователь остаётся на той же странице.
